リテラル（値）

速習を終えたところで、敢えて変数と代入をおさらいしましょう。

Pythonでは値を直接コードに書くことができます。

  • 値はそれだけで評価できる（エラーにならない）
  • 変数や関数なしで、その行に書くだけで意味を持つ
1
"1"
[1, 2, 3]
True
1.0

メモリ上での記述

上記の値はその行が実行された時に、メモリ上に値として記述されます（000001001 のようにメモリ上のスペースを使って記録される）。これらをリテラルと呼び、直接コード中に記述できる値の表現方法です。

変数とは

一方で、このように書くとエラーになります。なぜでしょうか。

a # NameError: name 'a' is not defined
  • 1 や "1" はリテラル（直接記述できる値）
  • a は値ではなく変数として認識される
  • 変数は値ではない — 値が書かれている場所を指す URL のようなもの（= アドレス）
  • print(変数) が値を表示するのは、ブラウザで URL を開くとページが表示されるのと同じ

代入の if（条件式）

if を使って代入する値を1行で切り替えられる。

score = 75
result = "合格" if score >= 60 else "不合格"
print(result)  # 合格

if-else で書くと以下と同じ意味：

if score >= 60:
    result = "合格"
else:
    result = "不合格"

いつ使う？

代入する値を条件で切り替えるだけの場面で使うとスッキリ書ける。 複雑な処理が必要なときは通常の if-else のほうが読みやすい。

変数の命名規則

変数として認識される単語のルールは以下のとおりです。

例外はありません。覚えましょう

  • 文字（a-z, A-Z）またはアンダースコア（_）で始まる
    • つまり数字始まりはダメ（NG例： 3200scores ）
    • 終わるのはOK（例：scores_of_3200）
  • 2文字目以降が文字、数字（0-9）、アンダースコアであること
  • 予約語（if / for / while など）ではないこと
a01    # 変数
_aa_aa # 変数
1a     # ダメ。最初が数字
continue # ダメ。予約語

代入

左辺に変数、右辺にリテラルを書いて = で結ぶと、「代入」が行われます。

a11 = 11
test_2 = "test2"

箱ではなく「看板」

a11 = 11 は「a11 という箱に 11 が入る」ではありません。

a11 は 11 が格納された住所を指す看板になる、というイメージです。

print(a11) はその看板が指す先の値を表示することを実装した関数なのです

変数 = 変数 の代入

変数同士を代入すると、左辺の看板が右辺の看板が指す先に書き換わります。

a11 = 11
a12 = 12
a11 = a12
print(a12) # 12
a11 = 13
print(a11) # 当然13
print(a12) # 11?13?

この結果は直感的に理解できますか？

リストの代入（罠）

代入するものが数値ではなく、リストの場合はどうでしょう。

odd1 = [1, 3]
odd2 = [5, 7]
odd1 = odd2 # odd1はodd2のアドレスを指すように指示
print(odd2) # [5, 7]
odd1[0] = 9 # odd2ではなくodd1を変更
print(odd2) # [9, 7]。 なぜodd2が変わっているの？

なぜ odd2 が変わっているのでしょうか？

第2回 演習③ の謎、解けますか？

第2回ショートカット演習③ で normalize(scores) を呼んだ後、山田が「合格」かつ「不合格者0人」に見えた理由が、この「リストの代入（罠）」で説明できます。

normalize() はリストの中身を同じアドレスのまま書き換えるため、ブロック②を実行した時点で scores 自体が補正済みになっています。ブロック③④は補正後のスコアを見ているため、55点の山田が消えていたのです。

id() で住所を確認（数値）

id() 関数で変数が指しているアドレスを調べられます。

a11 = 11
a12 = 12
print(id(a11)) # 2083050422832
print(id(a12)) # 2083050422864
a11 = a12
print(id(a11)) # 2083050422864。a12と同じ場所
a11 = 13
print(id(a11)) # 2083050422896。違う場所を指している。
  • 数値が代入されるたびに アドレスが変わっている
  • 数値は必ず新しいアドレスに新しい値として書き込まれる

id() で住所を確認（リスト）

リストでは、要素に代入してもアドレスが変わりません。

odd1 = [1, 3]
odd2 = [5, 7]
print(id(odd1)) # 2738928019264
print(id(odd2)) # 2738927989120
odd1 = odd2
print(id(odd1)) # 2738927989120
print(id(odd2)) # 2738927989120
odd1[0] = 9    # odd2ではなくodd1を変更
print(odd1)    # [9, 7]
  • odd1 = odd2 後、両者が同じアドレスを指している
  • だから odd1[0] = 9 が odd2 にも影響する

ミュータブル / イミュータブル

Pythonには、 - アドレスを使いまわす型（ミュータブル）

と

  • 必ず新しく作る型（イミュータブル）があります。

イミュータブル型としては、 int / float / str の3つだけ覚えておけば、まずはOKです。

分類 型
イミュータブル int / float / complex / bool / str / tuple / frozenset / bytes / None
ミュータブル list dict set など

覚え方

「変更可能か」ではなく、「アドレスを使いまわすか、新しく作るか」 で理解するほうが直感的です。

  • ミュータブル（mutable）：代入時にアドレスを共有する → list など
  • イミュータブル（immutable）：代入のたびに新しいアドレスが生まれる → int など

ミュータブルの罠を回避する：コピー

= で代入するとアドレスを共有してしまう。別のアドレスに複製したいときは .copy() を使う。

odd1 = [1, 3]

# NG: 同じアドレスを共有 → 両方が変わる
odd2 = odd1
odd2[0] = 9
print(odd1)  # [9, 3] ← 意図せず変わってしまう

# OK: .copy() で別アドレスの複製を作る
odd3 = odd1.copy()
odd3[0] = 99
print(odd1)  # [9, 3] ← 変わらない
print(odd3)  # [99, 3]

# スライス [:] でも同じ結果
odd4 = odd1[:]
print(id(odd1) == id(odd4))  # False（別アドレス）

.copy() vs [:]

どちらも同じ結果。.copy() のほうが意図が明確で読みやすい。

演習

  1. dict がミュータブルであることを id() を使って確認しましょう
  2. list と同じようにアクセスできる str がミュータブルかイミュータブルかを確認してください。
  3. 上記の例で odd1[0] のアドレスを確認して、どうなっているか見てみましょう（list はミュータブルですが、list の要素は int なのでイミュータブルのはず）

まとめ

  • 変数は値そのものではなく、値が置かれたアドレスを指す看板
  • int / float / str はイミュータブル — 代入のたびに新しいアドレスが生まれる
  • list / dict はミュータブル — 代入するとアドレスを共有するため意図せぬ書き換えに注意
  • 独立したコピーが必要なときは .copy() を使う
