リテラル（値）

速習を終えたところで、敢えて変数と代入をおさらいしましょう。

Pythonでは値を直接コードに書くことができます。

  • 値はそれだけで評価できる（エラーにならない）
  • 変数や関数なしで、その行に書くだけで意味を持つ
1
"1"
[1, 2, 3]
True
1.0

メモリ上での記述

上記の値はその行が実行された時に、メモリ上に値として記述されます（000001001 のようにメモリ上のスペースを使って記録される）。これらをリテラルと呼び、直接コード中に記述できる値の表現方法です。

変数とは

一方で、このように書くとエラーになります。なぜでしょうか。

a # NameError: name 'a' is not defined
  • 1 や "1" はリテラル（直接記述できる値）
  • a は値ではなく変数として認識される

変数の命名規則

変数として認識される単語のルールは以下のとおりです。

例外はありません。覚えましょう

  • 文字（a-z, A-Z）またはアンダースコア（_）で始まる
    • つまり数字始まりはダメ（NG例： 3200scores ）
    • 終わるのはOK（例：scores_of_3200）
  • 2文字目以降が文字、数字（0-9）、アンダースコアであること
  • 予約語（if / for / while など）ではないこと
a01    # OK。変数
_aa_aa # OK。変数
1a     # NG。最初が数字
continue # NG。予約語

代入

  • 変数 = リテラル と記述すると「代入」という操作がおこなわれる
    • 左辺は変数
    • 右辺はリテラルかオブジェクト
a = 1234
test_2 = "test2"

箱ではなく「看板」

  • a = 1234 は「a という箱に 1234 が入る」ではない
  • a は 1234 が格納された住所（アドレス）を指す看板になる
  • ただし、アクセス時はその看板自体ではなく住所が指す先にあるリテラルやオブジェクトが返る

変数 = 変数 の代入

  • 変数同士の代入は、左辺に右辺のアドレスを代入する
    • つまり、同じアドレスを指すようになる
a11 = 11
a12 = 12
a11 = a12 # a11とa12は同じアドレスを指す
print(a12) # 12

a11 = 13 # 別の値に変更
print(a11) # 当然13
print(a12) # ??? （同じアドレスを指しているはずだが？）
  • 結果： 12
  • 数値（int）は、宣言する度 新しいアドレス にリテラルを作る
  • つまり、5行目の時点でa11は別のアドレスが書かれている
  • よって、3行目で共有したアドレスはa12しか持たなくなる

リストの代入（罠）

数値でやった操作をリストでやると…

odd1 = [1, 3]
odd2 = [5, 7]
odd1 = odd2
odd1[0] = 9
print(odd2) # ???
  • 結果： [9, 7]
  • odd1[0] = 9 は odd1 を変えただけなのに、odd2 まで変化
    • つまり、3行目で共有したアドレスはずっと共有したまま

id() で住所を確認（数値）

id() 関数で変数が指しているアドレスを調べられます。

a11 = 11
a12 = 12
print(id(a11)) # 2083050422832
print(id(a12)) # 2083050422864
a11 = a12
print(id(a11)) # 2083050422864。a12と同じ場所
a11 = 13
print(id(a11)) # 2083050422896。違う場所を指している。
  • リテラルが代入されるたびに アドレスが変わっている
  • リテラルは必ず新しいアドレスに新しい値として書き込まれる

id() で住所を確認（リスト）

リストでは、要素に代入してもアドレスが変わりません。

odd1 = [1, 3]
odd2 = [5, 7]
print(id(odd1)) # 2738928019264
print(id(odd2)) # 2738927989120
odd1 = odd2
print(id(odd1)) # 2738927989120
print(id(odd2)) # 2738927989120
odd1[0] = 9    # odd2ではなくodd1を変更
print(odd1)    # [9, 7]
  • odd1 = odd2 後、両者が同じアドレスを指している
  • だから odd1[0] = 9 が odd2 にも影響する

ミュータブル / イミュータブル

Pythonにおいて変数が指せるValueには以下の２種類が存在

  • 代入時に必ず新しいアドレスに新たに作られる型（イミュータブル）
    • アドレスを意識しなくてもいい楽な型
    • int str bool 等
  • 代入時に現状のアドレスの中身を変更する型（ミュータブル）
    • ずっと同じところに居る
    • list、dict 等
分類 型
イミュータブル int / float / complex / bool / str / tuple / frozenset / bytes / None
ミュータブル list dict set クラス など

実例：関数がリストを書き換えてしまう

def double_all(data):
    for i in range(len(data)):
        data[i] = data[i] * 2
    return data

scores = [10, 20, 30]
result = double_all(scores)
print(result)  # [20, 40, 60]
print(scores)  # ???
  • 答えは [20, 40, 60] — scores も書き換わっている
  • result = double_all(scores) は新しいリストを作ったのではなく、同じリストの別名を作っただけ

ミュータブルの罠を回避する：コピー

  • リストを別のアドレスに新たに複製したいときは .copy() を使う。
    • この場面を残したいからセーブデータを一旦 別のスロットにするイメージ
    • 単に = で代入するとダメなケース
odd1 = [1, 3]

# NG: 同じアドレスを共有 → 両方が変わる
odd2 = odd1
odd2[0] = 9
print(odd1)  # [9, 3] ← 意図せず変わってしまう

# OK: .copy() で別アドレスの複製を作る
odd3 = odd1.copy()
odd3[0] = 99
print(odd1)  # [9, 3] ← 変わらない
print(odd3)  # [99, 3]

# （参考）全スライス [:] でも同じ結果
odd4 = odd1[:]
print(id(odd1) == id(odd4))  # False（別アドレス）

演習

  1. dict がミュータブルであることを id() を使って確認しましょう
  2. list と同じようにアクセスできる str がミュータブルかイミュータブルかを確認してください。
  3. 上記の例で odd1[0] のアドレスを確認して、どうなっているか見てみましょう（list はミュータブルですが、list の要素は int なのでイミュータブルのはず）

まとめ

  • 変数は値そのものではなく、値が置かれたアドレスを指す看板
  • int / str / bool 等はイミュータブル — 代入時に必ず新しいアドレスに作られる
  • list / dict 等はミュータブル — 同じアドレスの中身を変更するため意図せぬ書き換えに注意
  • 独立したコピーが必要なときは .copy() を使う
