【Collection１】 list（リスト型）

  • 最もよく使われるコンテナ
  • データは追加順に並んでいる
  • 他言語では配列やアレイと呼ばれる　
  • 変数名は、要素の名詞の複数形をつけることが多い（foodsなど）

list の初期化

  • まずは、右辺で list型ですよ と宣言しないと始まらない
  • ケースにより色々あるが [] が一般的

# 空のリスト（初期化）を作りたいときは以下のどちらか
empty1 = []
empty2 = list()

# カンマ区切りで初期値付きの初期化
scores = [85, 90, 78, 92, 88]

# print文で全体を表示
print(scores)   # [85, 90, 78, 92, 88]
print(empty1)   # []
print(empty2)   # []

# rangeから変換して初期化
odd_numbers = list(range(1, 11, 2))  # [1, 3, 5, 7, 9]

アクセス方法

  • [] に数字（インデックス or 添字）でアクセス
    • [1]、[500]
  • インデックスは先頭が 0（ゼロ）
    • なので、思ったより１つ少ない数でアクセス
      • ５番目の要素にアクセスしたい → [4]
print(scores[0])   # 85
print(scores[2])   # 78

# リストの長さ以上のインデックスを指定するとエラー
#print(scores[5])   # IndexError: list index out of range
# ↑ 長さは5だが、ゼロ始まりなので4が末尾

# マイナスを指定すると末尾からのアクセスになる
# ただし、リストの末尾は -0 ではなく -1
print(scores[-1])  # 88
print(scores[-5])  # 85

# リストの長さ+1のマイナスを指定するとエラー
print(scores[-6])  # IndexError: list index out of range

要素の追加（append）と結合

# 要素の追加は .append を使う（末尾に追加）
scores.append(95)
print(scores)  # [85, 90, 78, 92, 88, 95]

# 【注意】このような書き方はできない
score2 = scores.append(95)  # エラーにはならないが…
print(score2)  # None ← リストではない！

# リスト同士を接続（concat）することも可能
other_scores1 = [20, 30]
other_scores2 = [40, 50]
other_scores = other_scores1 + other_scores2
print(other_scores)  # [20, 30, 40, 50]

# これでも可
scores += other_scores
print(scores)  # [85, 90, 78, 92, 88, 95, 20, 30, 40, 50]

すべての要素へのアクセス（走査）

  • 基本的にfor inで使われる
# ① 基本系
for score in scores:
    print(f"{score}点")

# ② これも稀に使う
for i in range(len(scores)):
    print(f"学生{i + 1}は{scores[i]}点")

# ③ enumerate で ↑を同時に実現
for i, score in enumerate(scores):
    print(f"学生{i + 1}は{score}点")

どれを使う？

  • 要素だけ必要 → ①
  • 番号も必要 → ③（enumerate が一番スッキリ）
  • スライスなど index 操作が必要 → ②

演習問題（list）

student_scores = [85, 90, 92, 78, 92, 88]
  1. append を使って 100点 の学生を追加しよう
  2. 90点以上の学生の「学生番号」と「点数」を表示するプログラムを書こう
    • ヒント：for 文の中で if 文を使う
  3. 平均点を表示しよう
解答を見る 
student_scores.append(100)
for i, score in enumerate(student_scores):
    if score >= 90:
        print(f"学生{i + 1}は{score}点です")
print(sum(student_scores) / len(student_scores))

【Collection２】 dict（辞書型）

CC0: https://en.wikipedia.org/wiki/Fictionary#/media/File:Focus_definition_(Unsplash).jpg

dict の初期化

  • {} に キー: 値 のペアをカンマ区切りで書いて初期化
  • 変数名は name2score のような「x2y」形式が慣例
    • 2 は「→（to）」の意味
      • 「A2B」は「A to B」で「AからBへ変換」
# {"キー1": 値1, "キー2": 値2, ...} のように初期化
name2score = {"Yamada": 95, "Suzuki": 75}

# print で表示可能
print(name2score)        # {'Yamada': 95, 'Suzuki': 75}
print(type(name2score))  # <class 'dict'>

# 空の dict の初期化
dict_test1 = {}
dict_test2 = dict()

要素の追加・アクセス方法

  • ["キー"] でアクセス（list の [0] に相当）
  • 存在しないキーへの代入 → 追加、存在するキーへの代入 → 上書き
# ["キー"] でアクセス
print(name2score["Yamada"])  # 95

# 存在しないキーを指定するとエラー
print(name2score["Takahashi"])  # KeyError: 'Takahashi'

# 存在しないキーに代入すると追加
name2score["Takahashi"] = 80
print(name2score)  # {'Yamada': 95, 'Suzuki': 75, 'Takahashi': 80}

# 存在するキーに代入すると上書き
name2score["Suzuki"] = 70
print(name2score)  # {'Yamada': 95, 'Suzuki': 70, 'Takahashi': 80}

キーの存在確認（in 演算子）

in 演算子でキーや値の存在を確認できる。存在しないキーを指定する前に使うと KeyError を防げる。

name2score = {"Yamada": 95, "Suzuki": 75}

# キーが存在するか確認
if "Yamada" in name2score:
    print(name2score["Yamada"])  # 95

# 存在しないキーも安全にアクセスできる
if "Tanaka" in name2score:
    print(name2score["Tanaka"])
else:
    print("Tanaka は登録されていません")

# not in で「存在しない」を確認
if "Sato" not in name2score:
    print("Sato は未登録")

for 文での使い方（dict）

  • .keys() / .values() / .items() の3種類を使い分ける
  • 実務では .items() が最頻出
# すべてのキーを列挙
print(name2score.keys())    # dict_keys(['Yamada', 'Suzuki', 'Takahashi'])

# すべての値を列挙
print(name2score.values())  # dict_values([95, 70, 80])

# キーと値の組を列挙
print(name2score.items())   # dict_items([('Yamada', 95), ...])

# キーだけ使うパターン
for key in name2score.keys():
    print(f"{key}さんが居ます")

# キーと値を同時に使うパターン（最もよく使う）
for key, value in name2score.items():
    print(f"{key}さんは{value}点です")

演習問題（dict）

name2score = {"Yamada": 95, "Suzuki": 75, "Takahashi": 80}
  1. dict に100点を取った “Tanaka” さんを追加しよう
  2. Takahashi さんの点数を 92点 に訂正しよう
  3. 90点以上の学生の「名前」と「点数」を表示するプログラムを書こう
    • ヒント：items() を使って value を判定
  4. "Yamada" が dict に存在するか in を使って確認し、存在すれば点数を表示しよう
解答を見る 
name2score["Tanaka"] = 100
name2score["Takahashi"] = 92
for key, value in name2score.items():
    if value >= 90:
        print(f"{key}さんは{value}点です")
if "Yamada" in name2score:
    print(f"Yamadaさんの点数は{name2score['Yamada']}点です")

【Collection３】 tuple（タプル）

  • 組のこと。
    • 5つ組み等
  • 数学用語からきている
    • 例： 3tuple、ntuple
  • 要素が固定のリスト
  • （） で書かれる

PD: https://en.wikipedia.org/wiki/Coordinate_system#/media/File:3D_Spherical.svg

数学用語からきている

  • tupleは、組のこと
  • よく似た概念に集合（set）がある
性質 tuple（組） set（集合）
順序 あり
\((a,b) \neq (b,a)\)		 なし
\(\{a,b\} = \{b,a\}\)
重複 区別する
\((1,1,2) \neq (1,2)\)		 区別しない
\(\{1,1,2\} = \{1,2\}\)
定義 デカルト積（n個の集合の直積） 要素の集まり
用途 座標・関数の引数・関係 包含・和・積

Pythonでのtupleとset

  • tuple → 順序あり・重複あり・変更不可
  • set → 順序なし・重複なし（数学の集合に近い）

tuple の使いどころ

  • 要素数が固定のデータを表すのに使う
    • 座標 (x, y)、RGB (r, g, b) など
    • list よりメモリ効率がよい
  • 値の変更ができない（イミュータブル）
    • 意図せぬ書き換えを防げる
  • 関数から複数の値を返すときによく使う

tuple の初期化・アクセス

  • () にカンマ区切りで値を書いて初期化
    • 空の初期化もありうるが大抵は意味がない
  • アクセス方法は list と同じ（インデックス指定）
# 英語・数学・理科の点数を tuple で表現
score = (90, 85, 72)
print(score)          # (90, 85, 72)
print(type(score))    # <class 'tuple'>

# アクセスは list と同じ
print(score[0])   # 90
print(score[-1])  # 72（末尾からもOK）
print(score[3])   # IndexError: tuple index out of range

tuple と list の違い・制限

# 値の変更はエラー（listでは可能。タプルでは付加）
score[0] = 100  # TypeError: 'tuple' object does not support item assignment

# 値を追加できない（同じく）
score.append(100)  # error

# どうしても追加したい場合は、list へ一度変換すれば可能
# しかし、普通は使わない
score_list = list(score)
print(type(score_list))   # <class 'list'>
score_list.append(100)
score = tuple(score_list)
print(score_list)         # [90, 85, 72, 100]

なぜ tuple を使う？

変更させたくないデータを保護できる。list よりメモリ効率もよい。

よくある tuple の使われ方（関数の戻り値）

# リストの最小値と最大値を返す関数
def get_min_max_value(data):
    return (min(data), max(data))  # ← tuple で複数の値を返す

scores = [85, 90, 78, 92, 88]
answer = get_min_max_value(scores)
print(answer)  # (78, 92)

# 以下のように分けて受けることも可能
min_val, max_val = get_min_max_value(scores)

アンパック（unpacking）

tupleなどのイテラブルなクラスの各要素を複数の変数に一度に代入することをアンパックという。

point = (3, 7)
x, y = point      # x=3, y=7

変数の数と要素数が一致しないとエラーになる。

イテラブル（iterable）とは

イテレーション（iteration）は「コンテナから要素を1つずつ順番に取り出す操作」のこと。どの言語でも for 文の基本動作はこれ。イテラブルとは、イテレーションができるもの＝順番に要素を取り出せるもの。

型 イテラブル？
list ○
tuple ○
str ○（1文字ずつ）
range ○
dict ○（キーが順に出る）
int / float ✗

アンパックも for 文も、イテラブルであれば同じように使える。

実は dict で出てきていた tuple

dictのメソッド .items() はタプルの list を返している。だから key, value という2変数に代入できる。

print(name2score.items())
# dict_items([('Yamada', 95), ('Suzuki', 70), ('Takahashi', 80)])
#             ^^^^^^^^^^^^^^^ ← これが tuple！

for key, value in name2score.items():  # tuple を2変数に展開
    print(f"{key}さんは{value}点です")

演習問題（tuple）

def get_min_max(scores):
    return min(scores), max(scores)  # カッコなしでも tuple になる！

scores = [85, 90, 78, 92, 88]
  1. 関数を引数scoresでコールして、戻り値を変数 result で受け取る
  2. type(result) と値を print する
  3. low, high の2変数でアンパックして「最低: ○点, 最高: ○点」と表示
  4. result[0] = 100 と改ざんを試してみよう
解答を見る 
result = get_min_max(scores)
print(type(result), result)       # <class 'tuple'> (78, 92)

low, high = get_min_max(scores)
print(f"最低: {low}点, 最高: {high}点")

result[0] = 100  # TypeError: 'tuple' object does not support item assignment

まとめ

コレクションは複数のデータをまとめて管理するための型。

型 宣言 アクセス 追加・変更 主な用途
list [1, 2, 3] [i] ○ 順番あり・汎用
dict {"k": v} ["key"] ○ 名前でアクセス
tuple (1, 2, 3) [i] ✗ 座標・戻り値

発展：その他のコレクション型

現場でも競技プログラミングでもよく使われる。各自で調べよう。

  • deque — 先端への挿入・削除が高速（double ended queue）
  • defaultdict — キーが存在しない場合にデフォルト値を返す dict
  • heapq — 最小値の取り出しに強い（heap木で管理）
  • set — 重複が許されない。和集合などの論理演算も可能
A = {'a', 'b', 'c'}
B = {'b', 'c', 'd'}
print(A | B)  # {'a', 'b', 'c', 'd'}  ← 和集合
print(A & B)  # {'b', 'c'}            ← 積集合
print(A ^ B)  # {'a', 'd'}            ← 対称差
print(A - B)  # {'a'}                 ← 差集合

数値データには numpy の ndarray がよく使われる（後日扱う）。
