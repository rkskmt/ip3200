ショートカットキーを知るべき理由

最強の知的生産ツールであるPCを効率的に使ってタイパをあげよう。

マウス操作に対して、ショートカットキーで操作すると作業時間の大幅な短縮につながる。

例：文言を別の場所にコピーしたいとき

マウスの場合（8手順）

  1. キーボードからマウスに持ち替える
  2. マウスでコピーしたい文字列を選択する
  3. 右クリック
  4. コピーを選択
  5. マウスカーソルをペーストしたい場所にもってきてクリックする
  6. 右クリック
  7. ペーストを選択
  8. キーボードに手を戻す

ショートカットの場合（4手順）

  1. Shiftキーを押しながら矢印キーで選択範囲を指定する
  2. Ctrl+C キーでコピーバッファに格納
  3. ペーストしたい位置に矢印キーで移動
  4. Ctrl+Vでペースト

手順が半分になり、キーボードから手を離す必要もない。

スピードが倍化したとして、人生のトータルコピペ数を考えると馬鹿にならない差になる。

各種キーの名前

覚えておこう。

  • Ctrl キー（コントロール）
  • Shift キー（シフト）
  • Alt キー（オルタナティブの略なので「オルト」と読む）
  • Win キー（Windowsキー）

演習①：VS Codeで日本語入力

前提：日本語入力モード（IMEオン）で開始すること。

課題：VS Codeを開き、以下のコメントをそのまま入力しよう。

ゴール（これを入力する）：

# Alt（オルタナティブ）キーで行を移動できる
# Ctrl+/でコメントアウト、F5で実行

手順の例：Alt（オルタナティブ） の入力

  1. alt と打つ → F10（半角英数に変換）→ Alt → Enter で確定
  2. （ を入力
  3. おるたなてぃぶ と打つ → F7（カタカナに変換）→ オルタナティブ → Enter で確定
  4. ） を入力

漢字（移動、実行 など）は スペース で変換する。

演習②：カット＆ペースト＋字下げ

課題：順序と字下げが崩れたコードを ショートカットだけで 直そう。

初期状態：

score = int(input())
    if score >= 60:
print("不合格")
        print("合格")
    else:

ゴール：

score = int(input())
if score >= 60:
    print("合格")
else:
    print("不合格")

ヒント：

  • 行の移動は Alt + ↑↓ または Ctrl+X → Ctrl+V
  • 字下げは Ctrl + ]、戻すは Ctrl + [

演習③：コメントアウトでバグを追い詰める

課題：このプログラムを実行すると結果がおかしい。コメントアウトを使ってどのブロックが原因か突き止めよう。

コード（VS Codeに貼り付けて実行）：

scores = [85, 92, 78, 55, 65]
names = ["田中", "鈴木", "佐藤", "山田", "高橋"]

def normalize(data, threshold=60):
    for i in range(len(data)):
        if data[i] < threshold:
            data[i] = threshold
    return data

# ブロック①：受験者一覧
print("=== 受験者一覧 ===")
for i in range(len(names)):
    print(names[i], ":", scores[i], "点")

# ブロック②：補正後の平均点
print("=== 補正後の平均点 ===")
corrected = normalize(scores)
average = sum(corrected) / len(corrected)
print("平均：", round(average, 1), "点")

# ブロック③：合否判定
print("=== 合否判定 ===")
for i in range(len(names)):
    if scores[i] >= 60:
        print(names[i], "：合格")
    else:
        print(names[i], "：不合格")

# ブロック④：不合格者集計
print("=== 集計 ===")
failed = [s for s in scores if s < 60]
print("不合格者数：", len(failed), "人")
print("合格者数：", len(scores) - len(failed), "人")

おかしい点：①で山田は55点と表示されるのに、③では山田が合格、④では不合格者0人になっている。

手順：

  1. まず全体を実行して、おかしい箇所を確認する
  2. 怪しいブロックを 複数行まとめてCtrl+/ でコメントアウトして再実行
  3. どのブロックをコメントアウトしたら結果が正しくなるかを確認する
  4. 原因のブロックのどの行がバグか特定する

なぜこうなるの？

normalize() がリストを中で書き換えていることが原因です。 なぜそうなるのかのメカニズムは、変数と代入で詳しく解説します。

まとめ

  • ショートカットキーは手順を半分にし、生涯の作業時間を大幅に削減できる
  • Ctrl+C/V/Z/A などの基本ショートカットはほぼ全アプリで共通
  • Win+V（クリップボード履歴）・Win+Shift+S（スクリーンショット）も覚えておくと便利
  • VS Code では Alt+↑↓（行移動）・Ctrl+/（コメントアウト）・F5（実行）が特に役立つ
